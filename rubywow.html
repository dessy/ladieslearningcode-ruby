What is Ruby?

Ruby is a programming language.

Wait ... what's a programming language?

Let's backtrack a bit.

The Dilemma

Your computer speaks a language called machine language.

You can interact with you computer using your mouse and your keyboard.

You don't speak machine language, and machine language is really difficult for a person to speak, so learning isn't really an option.

However, you do need a way to tell your computer to do things.

The Solution

A programming language is a language that is relatively easy for you to learn, but can also be understood by the computer.

It's the middle-ground between English and machine language.

A programming language is made up of simple elements, and by combining those elements together, a programmer can write a set of instructions that the computer then breaks down and interprets as machine language.

You've just learned what programming is!

Programming is simply writing out sets of instructions that your computer can follow to do things.

Numbers and Words, Integers and Strings

In Ruby ...

Numbers without decimals are called integers. We just did some math with some integers.

Letters, words, and sentences are called strings. We tell Ruby that we are intending to use a string by wrapping it in quotes.

Objects, Objects, Everywhere!

You've already met Objects. That's a programming word!

Objects are everywhere in Ruby.

You can think of an Object in the Ruby language the same way you can think of a noun in the English language.

A noun is a particular person, place, or a thing.

The objects we worked with earlier, the Integer 99, or the String "ladieslearningcode", are things.

Objects are represented by a Class. That's another programming word!

Methods

next, odd?, and even? are all methods that you can call on an Integer.

capitalize, upcase, and reverse are all methods that you can call on a String.

When you call a method, you're telling the String "ladieslearningcode" to do something.

Methods are actions. You can compare methods in Ruby to verbs in English.

In English, you would tell Lisa Simpson to jump. In Ruby, you would call the jump method on lisa.

Giving Output Based on Input

A lot of a programmer's job, especially when working on web applications, is doing something based on the input the user provided.

Twitter, for example, disables the Tweet button if your tweet is over 140 characters, and keeps the button enabled if your tweet is 140 characters or shorter.

So far we know how to determine the length of the user's tweet, but we don't know how to tell them whether they can or cannot send their tweet, depending on it's length.

Conditional Logic

Programming is writing out sets of simple instructions for the computer to follow. Let's break down out tweet logic into simple instructions. Here is what we want our program to do:

if the tweet is greater than 140 characters, tell the user that they cannot send their tweet

if the tweet is less than or equal to 140 characters, tell the user that they can send their tweet

Your First Bug!

This is really a bug, since we're not actually taking into account all possible tweet lengths.

Most programming problems can be solved many different ways, and this bug is no exception.

We're going to look at three different solutions, and learn more about if statements as well as comparing objects along the way.

First (Worst) Solution

Since we're only missing one case, the case where the tweet is exactly 140 characters, we can just add another if statement.

Add this last if statement to the bottom of your file:

if tweet.length == 140
  puts "Tweet your heart out!"
end

The == operator checks that the value of the objects on either side of it are equal. Run your script again and notice that it catches the last case where a tweet is 140 characters long.

Refactoring Our Code

Programmers are always refactoring (re-strcturing, re-organizing) their code to make it cleaner and more readable. Right now is a great time to do that.

Three if statements are unnecessary. We can actually combine all of these if statements into one, like so:

if tweet.length > 140
  puts "Your tweet is too long!"
elsif tweet.length < 140
  puts "Tweet your heart out!"
elsif tweet.length == 140
  puts "Tweet your heart out!"
end

If the first if doesn't evaluate to true, then the next one is attempted, and so on. TODO: Talk about the details of this.

Second Solution

Our code is already better (and still has no bugs), but more refactoring can be done. Notice that the line puts "Tweet your hear out!" is repeated? This is unnecessary. Let's cut it out by replace the entire if statement with the code below: 

if tweet.length > 140
  puts "Your tweet is too long!"
elsif tweet.length <= 140
  puts "Tweet your heart out!"
end

The <= operator behaves the exact same way as the less than operator, but checks for equality as well.

As you may have guessed, checking for equality with greater than works exactly the same way, using the >= operator.

Third Time's a Charm

This is better, but look at the logic closely. Do we really even need to specify the second condition? Can't we just prevent the user from tweeting is their tweet is over 140 characters, and let them tweet in all other cases?

Yes we can! Replace your code again with the code below:

if tweet.length > 140
  puts "Your tweet is too long!"
else
  puts "Tweet your heart out!"
end

Run your code again, and it should function the exact same way!
