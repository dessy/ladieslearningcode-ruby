What is Ruby?

Ruby is a programming language.

Wait ... what's a programming language?

Let's backtrack a bit.

The Dilemma

Your computer speaks a language called assembly.

You can interact with you computer using your mouse and your keyboard.

You don't speak assembly, and assembly is really difficult for a person to speak, so learning isn't really an option.

However, you do need a way to tell your computer to do things.

The Solution

A programming language is a language that is relatively easy for you to learn, but can also be understood by the computer.

It's the middle-ground between English and assembly.

A programming language is made up of simple elements, and by combining those elements together, a programmer can write a set of instructions that the computer then breaks down and interprets as assembly.

You've just learned what programming is!

Programming is simply writing out sets of instructions that your computer can follow to do things.

Ruby knows the difference between a number and a word.

Numbers and Words, Integers and Strings

In Ruby ...

Numbers without decimals are called integers. We just did some math with some integers.

Letters, words, and sentences are called strings. We tell Ruby that we are intending to use a string by wrapping it in quotes.

You've just met your first two types of Objects. That's a programming word!

Objects, Objects, Everywhere!

Objects are everywhere in Ruby.

You can think of an Object in the Ruby language the same way you can think of a noun in the English language.

A noun is a person, place, or a thing.

The objects we worked with earlier, the Integer 2, or the String "ladieslearningcode", are things.

Objects are represented by a Class. That's another programming word!

Let's look at an example.

Classes

Going back to the English language and nouns ...

Lisa Simpson is a noun. Lisa Simpson is a type of cartoon character.

In Ruby, Lisa Simpson would be an object, and the Class representing Lisa would be CartoonCharacter.

Now back to an object we've used ...

"ladieslearningcode" is an object, and the Class representing that object is String. Want proof? Try this:

"ladieslearningcode".class
=> String

Simple String Manipulation 

> "ladieslearningcode".capitalize
=> "Ladieslearningcode"
> "ladieslearningcode".upcase
=> "LADIESLEARNINGCODE"
> "ladieslearningcode".reverse
=> "edocgninraelseidal"

Methods

capitalize, upcase, reverse, and length are all methods that you can call on a String.

When you call the method capitalize, you're telling the String to do something.

Methods are actions. You can compare methods in Ruby to verbs in English.

In English, you would tell Lisa Simpson to read a book. In Ruby, you would call the read_book method on lisa.

Does an Integer Behave the Same Way? 

> 5.capitalize
NoMethodError: undefined method `capitalize' for 5:Fixnum

Oops. Capitalizing the number 5 just doesn't make sense.

Let's look at that error message. A developer (even the best of the best) spends a lot of time reading error messages. NoMethodError.

Putting it all together

Some methods don't make sense on certain objects.

The capitalize method makes sense on the String "ladieslearningcode" but does not make sense on the Integer 5.

That's why 5 does not respond to the method capitalize.
